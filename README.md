# 📊 Excel Парсер карточек 1С

Приложение для парсинга Excel файлов (выгрузок из 1С) с накоплением данных и экспортом в CSV. Поддерживает автоматическое определение структуры файлов с объединенными ячейками.

## 🚀 Возможности

- **Поддержка форматов**: `.xls` и `.xlsx`
- **Накопительный режим**: загрузка нескольких файлов с накоплением данных
- **Автоопределение структуры**: умное определение столбцов с датами, плательщиками и суммами
- **Поддержка объединенных ячеек**: корректная обработка файлов с объединенными F-G и G-H
- **Предпросмотр данных**: просмотр первых 10 записей перед экспортом
- **Экспорт в CSV**: сохранение всех накопленных данных в CSV с выбором места сохранения
- **Русская локализация**: полная поддержка кириллицы

## 📋 Требования

- Python 3.12+
- UV (менеджер пакетов)

## 🔧 Установка

### 1. Клонирование репозитория

```bash
git clone <your-repo-url>
cd monthly_report
```

### 2. Установка UV (если не установлен)

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
source $HOME/.local/bin/env
```

### 3. Установка зависимостей

```bash
uv sync
```
Или вручную:
```bash
uv add nicegui pandas openpyxl xlrd
```

## 🏃‍♂️ Запуск

```bash
uv run python main.py
```
После запуска приложение будет доступно по адресу: http://localhost:8080

## 📖 Использование

### Загрузка файлов
Нажмите на область загрузки или перетащите Excel файл

Приложение автоматически определит структуру и извлечет данные:
- 📅 Дата (столбец A)
- 👤 Плательщик (столбцы D или E)
- 💰 Сумма (столбцы F, G или H, в зависимости от структуры)
- 📁 Источник (имя файла)

### Накопление данных
- Каждый новый файл добавляет записи к уже существующим
- В таблице отображаются все накопленные данные
- В верхней панели видно количество загруженных файлов и общее число записей

### Управление данными
- 🗑️ Очистить все данные - удаляет все накопленные записи
- 👁️ Предпросмотр данных - показывает первые 10 записей в диалоговом окне
- 📥 Экспорт всех данных в CSV - сохраняет все записи в CSV файл с выбором места сохранения

## 🧩 Структура данных
Приложение извлекает данные из следующих столбцов:

|Данные|Столбцы|Примечание|
|---|---|---|
|Дата|A|Начиная с 10 строки|
|Плательщик|D или E|Автоматическое определение
|Сумма|F, G или H|Автоопределение с учетом объединений

## 🔍 Автоопределение структуры
Приложение умно определяет структуру каждого файла:
- Анализирует первые 5 строк данных
- Определяет наличие объединенных ячеек (F-G, G-H)
- Выбирает столбец с суммами на основе статистики чисел
- Поддерживает файлы с разной структурой

## 📁 Пример файла
Файл должен иметь следующую структуру:
- Строка 10 и далее - данные
- Столбец A - даты
- Столбцы D/E - плательщики
- Столбцы F/G/H - суммы (с возможными объединениями)

## 🛠️ Технологии
- NiceGUI - веб-интерфейс
- Pandas - обработка данных
- OpenPyXL - чтение .xlsx файлов
- xlrd - чтение .xls файлов

## 📝 Формат экспорта CSV
Экспортируемый CSV файл содержит следующие колонки:
- date - дата платежа
- buyer - плательщик
- pay_sum - сумма
- source_file - исходный файл

Кодировка: UTF-8 with BOM (utf-8-sig) для корректного отображения в Excel

## ❗ Возможные проблемы и решения

### Ошибка "xlrd not installed"

```bash
uv add xlrd
```
### Ошибка "openpyxl not installed"
```bash
uv add openpyxl
```
### Русские буквы в CSV отображаются некорректно
Приложение использует кодировку utf-8-sig, которая корректно отображается в Excel. Если проблема сохраняется, откройте CSV через:
- Excel: Данные → Из текста/CSV → Выбрать UTF-8
- LibreOffice: автоматически определит кодировку

### Файл не распознается
Проверьте консоль на наличие отладочной информации. Приложение выводит:
- Размер файла
- Структуру столбцов
- Определенные настройки
